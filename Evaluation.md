# Evaluation

## Цели

Основная цель — построить систему discovery которая позволяет находить релевантные таблицы и колонки по ключевым словам. Система должна работать с несколькими источниками данных и возвращать ранжированные результаты с метаданными.

## Метрики

### 1. Precision@1
Насколько первый результат релевантен запросу.

**Как считается:** для каждого тестового запроса проверяем совпадает ли первый результат с ожидаемым.

### 2. MRR (Mean Reciprocal Rank)
На какой позиции находится первый правильный ответ.

**Как считается:** `1 / позиция_первого_правильного_ответа`

### 3. Recall
Сколько релевантных результатов из всех возможных нашла система.

## Тестовые запросы

| Запрос | Ожидаемый результат | Тип |
|--------|---------------------|-----|
| `country` | users → country | column |
| `France` | users → country | value |
| `cuntry` | users → country | fuzzy |
| `age` | users → age | column |
| `income` | customer_sample_500 → Income | column |
| `xyz123` | пустой список | — |

## Результаты

| Запрос | Позиция правильного ответа | Score | MRR |
|--------|---------------------------|-------|-----|
| `country` | 1 | 0.9 | 1.0 |
| `France` | 1 | 0.5 | 1.0 |
| `cuntry` | 1 | 0.74 | 1.0 |
| `age` | 1 | 0.9 | 1.0 |
| `income` | 1 | 0.9 | 1.0 |
| `xyz123` | — | — | — |

**Средний MRR: 1.0**
**Precision@1: 5/5 = 1.0**

## Анализ

**Сильные стороны:**
- Точный поиск по именам колонок работает отлично
- Fuzzy поиск находит результаты даже при опечатках
- Поиск по sample values позволяет искать по реальным данным
- Чувствительные колонки автоматически помечаются

**Ограничения:**
- Поиск работает только по именам и sample values — семантический поиск недоступен
- Запрос "где хранится местоположение пользователя" не найдёт колонку `country`
- Sample values ограничены 5 значениями — редкие значения не индексируются

## Next Steps

**1. Семантический поиск через эмбеддинги**
Добавить `sentence-transformers` для векторизации имён колонок и запросов. Гибридный скоринг: `score = 0.6 * fuzzy + 0.4 * cosine_similarity`. Это решит проблему семантических запросов.

**2. Фильтры**
Добавить фильтрацию по источнику, типу данных, дате индексации:
```
GET /search?q=age&source=netflix_db&type=column
```

**3. Инкрементальная индексация**
Отслеживать `last_modified` файлов и переиндексировать только изменившиеся источники.

**4. Логирование**
Добавить логирование запросов для анализа популярных поисков и улучшения ранжирования.
